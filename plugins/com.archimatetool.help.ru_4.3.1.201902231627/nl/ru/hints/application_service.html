Сервис программ определяется как видимая откуда-то извне единица работы программ, обеспечиваемая одной или многими программами, выставленная через хорошо определенные интерфейсы, и значимая для окружения.

Сервис программ выставляет функционал программ во внешнее окружение. Этот функционал может быть доступен через один или более интерфейсов программы

Сервис программ реализуется одним или более функционалом программы, которые выполняются программами. Он может требовать, использовать и создавать данные.

Сервис обработки данных может быть использован процессом, практикой, коллегиальным процессом, или функционалом программы.

Функционал программы может реализовать сервис программ. Интерфейс программы может быть назначен для сервиса программ. Сервис программ может иметь доступ к данным.

Имя сервиса программ должно быть предпочтительно отглагольным существительным, имя также может содержать в себе слово &qout;сервис&unquot;.

Категория:

"Софт". Работы.

Примеры:

Обработка транзаций, Платёжный сервис, Сервис ведения клиентов.

Типовые отношения:

Использует - процесс, практику, коллегиальный процесс, функционал программы
Реализован - функционалом программы
Назначен - для интерфейса программы
Доступ - к данным
